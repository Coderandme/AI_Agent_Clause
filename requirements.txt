# Dependencies for the Hugging Face Space (Gradio SDK).
#
# DELIBERATELY MINIMAL AND DELIBERATELY UNPINNED. Read this before "tidying" it.
#
# The Gradio SDK does not simply install this file. It force-installs its own
# `gradio[oauth,mcp]==<sdk_version>` on top, and the `mcp` extra carries a hard constraint of
# `pydantic<=2.12.5`. The first version of this file was exported straight from the lockfile with
# exact pins on everything — including transitive packages we never import — which pinned
# `pydantic==2.13.4` and collided head-on with that. The build died with ResolutionImpossible.
#
# So: list only what we actually import, and let pip resolve the shared packages against HF's
# constraints rather than against ours. We do not own this environment. We are a guest in it.
#
# Notably ABSENT, and correctly so:
#   gradio, gradio-client    HF installs these itself, from `sdk_version` in the README frontmatter.
#   pydantic, pydantic-core  pulled in transitively. Pinning them is exactly what broke the build.
#   fastapi, uvicorn         Gradio brings its own server. The FastAPI app in api/clause/app.py is
#                            for the eventual Next.js frontend and is not what runs here.
#   boto3, jinja2            R2 storage and memo templates. Neither is on the Space's code path yet.
#
# The exact, fully-pinned dependency set lives in api/uv.lock, and that is what the Dockerfile
# installs. That one is reproducible. This one has to play nicely inside someone else's image.

asyncpg>=0.30           # Postgres: the usage ledger and the spend caps
openai>=2.0             # the agent
pymupdf>=1.24           # parse -> text + per-page character offsets
rapidfuzz>=3.10         # quote verification: locating a candidate span
pyyaml>=6.0             # the rule library is data, not code
pydantic-settings>=2.6  # config from the environment
